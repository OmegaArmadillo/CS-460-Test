CS460 Project 1: Basic Webpage Creation
This page contains information about my Project 1.
The purposes of this project was to do things like learn how to create a repository, learn how to interact with the repository using the command line, create a webpage with html and css along with using Bootstrap.
We also had a list of requirments for our webpage.
For my website I had the idea of creating a website for a traveling agency.
The pages I created included a home page, a page that shows different locations, and a page that shows the travel rates.

Creating the Repository
This part required me to create a GitHub repository and learn how to do things such as being able add and commit files from the command line.
At first I used this page to learn how to start.
I then followed these steps.
  1. Created a new repository manually in GitHub.
  2. Used the command git init to initialize the repository in the correct folder.
  3. Used the command git add . to add the files to stage them for commiting.
  4. Used the command git commit -m "First commit" to commit the files that I staged.
  5. Found my repository's url in the setup page of my repository. (https://omegaarmadillo.github.io/CS-460-Test/)
  6. Used the command git remote add origin https://omegaarmadillo.github.io/CS-460-Test/ to set a new remote.
  7. Used the command git remote -v to verify the remote.
  8. Finally I used the command git push origin master to push the code to the repository.
I also learned about the uses of the pull command.
I had to use it after I had edited and deleted some files in my repository and tried to push files.
I then got the error "Updates were rejected because the remote contains work that you donot have locally".
After I got this error I did some research and learned I had to use git pull origin master to fix the issue before I could push to the repository.

Creating the Webpage
For this part there were these were the things that were required
  1. You must use Bootstrap (v4) for the layout of all pages and must demonstrate both single column layout and two or more column formatting
  2. You must have a separate CSS file (e.g. styles.css) in which you write some of your own classes and that you use somewhere
  3. You'll need a navigation bar or menu that contains links to all your pages
  4. You need to have at least one table and at least two kinds of lists: ol, ul and dl

3. Create a navigation bar
For this I decided to use an unordered list as a navigation bar so i could fufill two requirements.
This is my code
<div class = "navMenu">
<ul class = "navMenuList">
<li class = "navMenuListItem"> <a href = "HW1_Sights.html"> Sights </a> </li>
<li class = "navMenuListItem"> <a href = "HW1_Rates.html"> Rates </a> </li>
</ul>
</div>

These are the styles I created
.navMenu
{
text-align: center;
color: white;
background-color: GoldenRod;
height: 40px;
}
.navMenuList
{
list-style-type: none;
}
.navMenuListItem
{
float: left;
margin:0 0 0 50px;
font-size: 20px;
}

I decided to use this many classes so I could have a wide range of control when creating the website.
I found that using a list was easier than making a bunch of divs as I din't have to worry about messing with the width of each of the divs.
All I had to do was use the float:left to achieve a similiar look.
